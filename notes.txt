Azure Ollama deployment notes

1. Login
az login

2. Build and push to ACR
From C:\Users\loreaa\VS_CODE\SELF-HOSTED-FOUNDRY:
az acr build --registry 0a20b02aigovernanceacr --image self-hosted-ollama:v1 --source-acr-auth-id [caller] .

3. Verify the pushed tag
az acr repository show-tags --name 0a20b02aigovernanceacr --repository self-hosted-ollama --output table

4. Build an amd64 image for Azure Container Instances
ACI rejected the original arm64 image, so this amd64 tag is required for deployment:
az acr build --registry 0a20b02aigovernanceacr --image self-hosted-ollama:v1-amd64 --platform linux/amd64 --source-acr-auth-id [caller] .

5. Verify the amd64 tag
az acr repository show-tags --name 0a20b02aigovernanceacr --repository self-hosted-ollama --output table

6. Register Azure Container Instances provider if needed
az provider register --namespace Microsoft.ContainerInstance
az provider register --namespace Microsoft.ContainerInstance --wait
az provider show --namespace Microsoft.ContainerInstance --query "{namespace:namespace,registrationState:registrationState}" -o json

7. Create a managed identity for ACI to pull from the private ABAC-enabled ACR
az identity create -g rg-loreaa-test13 -n mi-aci-ollama-pull --location eastus2 --query "{id:id,principalId:principalId,name:name}" -o json

8. Grant the identity pull access to the registry
az role assignment create --assignee-object-id $(az identity show -g rg-loreaa-test13 -n mi-aci-ollama-pull --query principalId -o tsv) --assignee-principal-type ServicePrincipal --scope $(az acr show -n 0a20b02aigovernanceacr --query id -o tsv) --role "Container Registry Repository Reader" -o json

9. Deploy to Azure Container Instances
$identityId = az identity show -g rg-loreaa-test13 -n mi-aci-ollama-pull --query id -o tsv; az container create -g rg-loreaa-test13 -n aci-self-hosted-ollama --image 0a20b02aigovernanceacr.azurecr.io/self-hosted-ollama:v1-amd64 --assign-identity $identityId --acr-identity $identityId --cpu 2 --memory 8 --ports 11434 --ip-address Public --os-type Linux --restart-policy Always -o json

10. Check deployment status
az container show -g rg-loreaa-test13 -n aci-self-hosted-ollama --query "{name:name,provisioningState:provisioningState,state:containers[0].instanceView.currentState.state,ipAddress:ipAddress.ip,ports:ipAddress.ports}" -o json

11. View logs
az container logs -g rg-loreaa-test13 -n aci-self-hosted-ollama

12. Test the public endpoint
Invoke-WebRequest -UseBasicParsing http://<public-ip>:11434/api/tags | Select-Object -ExpandProperty Content

13. Current live endpoint
Public IP: <public-ip>
Port: 11434

14. Notes
- The ACR 0a20b02aigovernanceacr uses AbacRepositoryPermissions, so [caller] is required for az acr build.
- Azure Container Instances needed a linux/amd64 image, so deploy v1-amd64 rather than v1.
- The endpoint is public and unauthenticated.
- The model is currently stored inside the container filesystem, so a redeploy can require re-pulling the model unless persistent storage is added.
